# 작업지침 (AI·사람 공통 규칙)

이 파일은 사용자가 작업 중에 내린 지침을 모은 것이다. **어떤 PC·어떤 세션에서 작업하든
이 규칙을 그대로 따른다.** 새 지침이 생기면 여기에 추가한다.

---

## 0. 대원칙
- 대상은 **`합격선 최종모음`** 폴더뿐. (`합격선 모음` 등 다른 폴더는 보지 않는다.)
- **원본 파일은 절대 수정하지 않는다.** 읽기 전용.
- 최종 진실은 **`검증_data.csv`** 하나. 여기에 사람이 확인한 데이터만 쌓는다.
- 속도보다 **정확하고 보기 편한 표**가 우선.

## 1. 읽기 방식 — 파서 맹신 금지
- 자동 일괄 파서를 그대로 믿지 않는다. 과거 hwp 컬럼 밀림/컬럼 매핑 오류로
  엉뚱한 합격선이 들어간 사고가 있었다.
- 절차: **PDF 텍스트를 직접 추출(pdfplumber) → 컬럼을 정확히 매핑 → 의심되면 해당 행을
  PyMuPDF로 확대 렌더해 눈으로 교차검증 → `검증_data.csv`에 입력.**
- 추출값과 눈읽기가 다르면, 평균×과목수=총점 같은 내부 정합성으로 판정한다
  (실측상 PDF 텍스트 추출이 눈읽기보다 정확했다).
- **한 폴더(지역 × 공무원/교행)를 끝낼 때마다 사용자 확인을 받고** 다음 폴더로 넘어간다.

## 2. 포함/제외
- **경력경쟁은 모두 제외.** `공개경쟁`만 수록.
  - **인천 한정** 경력경쟁 = 직렬 **수의·수의연구·운전** + 대상 **북한이탈주민·보훈청추천**.
    시설관리(일반)는 공개경쟁으로 **포함**.
- **7급은 별도로 분리**해서 본다(직급 필터로 구분; 데이터에는 직급 `7급`으로 그대로 기입).

## 3. 합격선 표기
- `합격선기준` = `과목평균`(100점 만점) 또는 `총점`.
- 둘 다 제공되는 지역(인천 등)은 **합격선=과목평균**, 총점은 `비고`에 `총점OOO`.
- 합격자가 없으면 합격선은 빈칸, `비고`에 `미선발`.

## 4. 임용예정기관 (시·군별)
- 시·군마다 커트라인이 다른 지역(인천·경기·충남·강원 등)은 **임용예정기관 칸**에 기관명을
  넣고 **시·군별로 행을 모두 분리**해서 표시한다. (예: 인천 시·구 / 강화군 / 옹진군)

## 5. 직류 네이밍
- **직류 이름을 합치거나 바꾸려면 반드시 사용자에게 먼저 묻는다.**
  (예: "행정"과 "일반행정"을 같은 것으로 통합 — 임의로 하지 말 것.)
- 직렬(류) 묶음 표기 해석: `행정9급(일반행정_일반)` = 직렬`행정` + 직급`9급` + 직류`일반행정` + 대상`일반`.
  대상 매핑: 일반→일반, 장애→장애인, 저소득→저소득층.

## 6. 계산 컬럼
- 경쟁률(접수/선발), 응시율(응시/접수×100), 경쟁률(응시/선발)을 **직접 계산**해 채운다.

## 7. HTML(`합격선_관리_v2.html`)
- 다크모드 X, **깔끔한 라이트 테마**.
- 지역별/직렬별/직류별로 볼 수 있는 **필터**, 응시율 표시, 헤더 너비 통일.
- **직급 필터 기본값 = 9급.**
- "출처" 칸은 원본 파일로 바로 열리는 **하이퍼링크**(`file://`).
- `검증_data.csv` 만 고치고 `python build_v2.py` 로 다시 생성한다.
