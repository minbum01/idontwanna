# 합격선 통합 산출 (idontwanna)

전국 17개 지역 지방공무원·교육행정 **필기 합격선(커트라인)** 데이터를 한 곳에 모아
검증된 표(HTML)로 보는 프로젝트.

핵심 산출물은 **`검증_data.csv`(사람이 확인한 데이터)** 와 이를 렌더링한
**`합격선_관리_v2.html`** 두 개다. 나머지 `parse_*.py`, 옛 CSV/HTML 등은 과거 자동파싱
파이프라인의 흔적이며 **현재는 사용하지 않는다**(아래 "작업 방식" 참고).

---

## 폴더 구조 (다른 PC에서 동일하게 맞출 것)

```
이민범 개발/합격선/
├─ 합격선 최종모음/          ← 원본 파일(읽기 전용, 절대 수정 금지). 별도로 복사해 둘 것
│   ├─ 01 서울 공무원/  02 서울 교행/ ...  (지역 × 공무원/교행, 약 250개 파일)
│   └─ ...
└─ _통합산출/                ← 이 깃 저장소
    ├─ 검증_data.csv         ← ★ 사람이 확인한 최종 데이터 (유일한 진실)
    ├─ build_v2.py           ← 검증_data.csv → 합격선_관리_v2.html 생성기
    ├─ 합격선_관리_v2.html    ← 결과물(브라우저로 열기)
    ├─ README.md
    └─ 작업지침.md           ← ★ AI/사람이 작업할 때 반드시 지킬 규칙
```

- 저장소(`_통합산출`)는 **원본 파일을 포함하지 않는다**(용량·수정금지 때문).
  다른 PC에서는 `합격선 최종모음` 폴더를 위 구조대로 **이 저장소의 상위 폴더에** 놓아야
  HTML의 "출처" 하이퍼링크(`file://`)가 동작한다.
- 원본 폴더 위치가 다르면 환경변수로 지정: `HAPGYEOK_SRC=경로 python build_v2.py`

## 환경 설정

Python 3.10+ (Windows 기준, 다른 OS도 동일).

```bash
pip install pdfplumber PyMuPDF Pillow openpyxl olefile
```

| 패키지 | 용도 |
|---|---|
| `pdfplumber` | PDF 표 텍스트 추출 (메인) |
| `PyMuPDF`(fitz) + `Pillow` | PDF→이미지 렌더(눈으로 교차검증할 때) |
| `openpyxl` | xlsx 읽기 |
| `olefile` | hwp(HWP5) PrvText 미리보기 텍스트 추출 |

## 사용법

```bash
# 결과 HTML 다시 만들기 (검증_data.csv를 고친 뒤 실행)
python build_v2.py
# → 합격선_관리_v2.html 생성. 브라우저로 열면 됨.
```

`합격선_관리_v2.html` 은 단일 파일(데이터 내장)이라 더블클릭으로 바로 열린다.
필터: 지역 / 시험종류 / 연도 / 임용예정기관 / 직급(기본 9급) / 직렬 / 직류 / 대상 + 검색.

---

## 데이터 스키마 (`검증_data.csv`)

```
지역, 시험종류, 연도, 회차, 시험구분, 임용예정기관, 직군, 직렬, 직류, 직급, 대상,
선발예정인원, 접수인원, 경쟁률(접수/선발), 응시인원, 응시율(%), 경쟁률(응시/선발),
필기합격인원, 합격선, 합격선기준, 양성평등합격선, 원본파일명, 비고
```

- **합격선기준**: `과목평균`(100점 만점 평균) 또는 `총점`. 인천처럼 둘 다 나오면
  **합격선=과목평균**, 총점은 `비고`에 `총점OOO`로.
- **임용예정기관**: 시·군별로 커트라인이 다른 지역(인천·경기·충남·강원 등)은 기관별로 행 분리.
- **경쟁률/응시율**: 원본에 없으면 직접 계산해서 채움.
- **시험구분**: `공개경쟁`만 수록. `경력경쟁`은 모두 제외(작업지침 참고).

## 작업 방식 (중요)

자동 파서를 신뢰하지 않는다. 과거에 hwp 컬럼 밀림·컬럼 매핑 오류로 잘못된 합격선이 들어간
적이 있어, **PDF 텍스트를 직접 추출(pdfplumber)해서 컬럼을 정확히 매핑하고, 의심되면
PyMuPDF로 해당 행을 확대 렌더해 눈으로 교차검증**한 뒤 `검증_data.csv`에 넣는다.
한 폴더(지역×구분)를 끝내면 사용자가 확인하고 다음 폴더로 넘어간다.

자세한 규칙은 **[작업지침.md](작업지침.md)** 참고.
